글을 발행했는데 구글 검색에 안 뜬다면, 대부분은 콘텐츠 문제가 아니라 색인 단계에서 막힌 겁니다.

서치콘솔을 열어보면 이런 메시지가 보일 거예요.

- 색인 안 됨 / 제외됨
- Discovered - currently not indexed
- Crawled - currently not indexed
- 색인 생성 요청을 눌러도 변화 없음

이 글에서 각 상태별로 어디를 확인하고 뭘 고쳐야 하는지 정리했습니다.

---

## "발견(Discovered)"과 "크롤링(Crawled)"은 다릅니다

### Discovered - currently not indexed
구글이 URL을 찾긴 했는데 아직 크롤링을 안 한 상태입니다. 사이트 부하를 고려해 크롤을 미루거나, 우선순위가 낮다고 본 경우예요.

### Crawled - currently not indexed
크롤링은 했는데 색인할지 결정을 안 한 상태입니다. 콘텐츠 품질이나 중복 문제일 가능성이 높아요.

상태에 따라 해결 방법이 다릅니다.

---

## 색인 안 될 때 가장 먼저 확인할 것들

아래 7가지 중 문제가 있는지 먼저 보세요.

### (1) URL 검사로 현재 상태 확인

서치콘솔에서 URL 검사 → 색인 가능한지, 차단되는지 확인합니다.
필요하면 색인 생성 요청을 할 수 있어요 (반영까지 1~2주 걸립니다).

### (2) robots.txt 확인

`yourdomain.com/robots.txt`에 접속해서 확인하세요.

- `Disallow: /`가 있으면 전체가 막힙니다
- 특정 경로만 막아도 해당 글이 색인 안 됩니다

### (3) noindex 태그 확인

페이지 소스(Ctrl+U)에서 `<meta name="robots" content="noindex">`가 있는지 보세요.
404 페이지에만 넣으려다 전체 템플릿에 들어가는 실수가 흔합니다.

### (4) canonical 태그 확인

`<link rel="canonical" href="..." />` 태그가 현재 페이지 URL과 일치하는지 확인하세요.
다른 URL을 가리키면 구글은 그쪽을 원본으로 보고 이 페이지는 제외할 수 있습니다.

### (5) 중복 URL 정리

http/https, www/non-www, 슬래시 유무가 섞이면 중복으로 봅니다.
표준 URL 하나로 통일하고 나머지는 301 리다이렉트 처리하세요.

### (6) 사이트맵 확인

sitemap.xml을 열어서 해당 글 URL이 실제로 들어있는지 확인하세요.
서치콘솔에 사이트맵이 제출되어 있는지도 체크합니다.

### (7) 내부링크 추가

홈/카테고리/관련 글에서 새 글로 링크를 3~5개 연결하세요.
외부 링크가 없어도 내부 연결만 잘 되어 있으면 색인 속도가 빨라집니다.

## 상태별 해결 방법

### Discovered 상태 해결법

크롤링 우선순위를 올리는 게 목표입니다.

#### 효과 좋은 방법 5가지

**내부 링크 추가**
- 홈페이지 최신글 영역에 노출
- 카테고리 페이지에서 연결
- 관련 글 본문에 링크 삽입
- 인기 글 위젯에 추가

**사이트맵 재제출**
- sitemap.xml에 새 글이 들어갔는지 확인
- 서치콘솔에서 재제출
- lastmod 날짜가 최신인지 체크

**서버 응답 확인**
- 5xx 오류나 타임아웃이 있으면 크롤이 미뤄집니다
- 페이지 로딩 속도 3초 이내 유지
- Core Web Vitals 확인

**불필요한 페이지 정리**
- 태그 페이지, 중복 카테고리가 많으면 크롤 예산이 분산됩니다
- 불필요한 태그 페이지는 noindex 처리
- 페이지네이션은 canonical로 정리

**색인 생성 요청**
- URL 검사에서 1회만 요청하세요
- 여러 번 해도 효과 없습니다

---

### Crawled 상태 해결법

색인할 이유를 만들어주는 게 목표입니다.

#### 얇은 글 문제 해결

이 중에서 2개만 추가해도 달라집니다.

- 단계별 해결 과정 추가 (스크린샷 포함)
- "자주 하는 실수" 섹션 추가
- 한눈에 보는 체크리스트 테이블
- FAQ 5개 이상
- 구체적인 상황별 예시
- A vs B 비교 콘텐츠

**최소 기준**
- 본문 1,500자 이상 (이미지 포함시 1,000자)
- H2 제목 4개 이상
- 실용적인 정보 (정의만 나열 X)

#### 검색 의도에 맞추기

검색하는 사람이 궁금한 걸 소제목으로 바로 넣으세요.

예시:
- "색인 생성 요청했는데 왜 안 되나요?"
- "사이트맵 제출했는데도 왜 안 되나요?"
- "신생 블로그는 얼마나 기다려야 하나요?"

#### 중복 문제 해결

같은 내용이 다른 URL로 있으면 구글은 하나만 색인합니다.

확인할 것:
- URL 파라미터 정리 (utm, ref 등)
- 프린트 페이지 noindex 처리
- AMP/모바일 버전 canonical 확인

## 자주 묻는 질문

### 색인 생성 요청했는데 왜 안 되나요?

요청은 바로 노출을 보장하지 않습니다. 구글에게 다시 확인해달라는 신호일 뿐이에요.
반영까지 보통 1~2주 걸립니다. 같은 URL을 여러 번 요청해도 효과 없으니 1회만 하세요.

### 사이트맵 제출했는데도 색인이 안 돼요

사이트맵 제출이 색인을 보장하진 않습니다.

추가로 확인할 것:
- sitemap에 해당 URL이 실제로 있나?
- robots.txt나 noindex로 차단 안 했나?
- 내부 링크가 3개 이상 있나?
- 중복 URL은 없나?

### 페이지가 발견됨인데 크롤링이 안 되는 이유는?

구글이 크롤을 미루는 상태입니다.

해결 순서:
1. 내부링크 강화
2. 사이트맵 재제출
3. 서버 안정성 확인 (5xx 오류, 느린 응답)
4. 불필요한 페이지 정리

### 신생 사이트는 원래 느린가요?

네, 초기엔 그럴 수 있습니다. 구글이 사이트 신뢰도를 평가하는 시간이 필요해요.

신생 사이트에서 해야 할 것:
- 내부 링크 구조 탄탄하게
- 사이트맵 최신 상태 유지
- 중복 URL 정리
- 꾸준한 발행 (주 2~3회)

### 색인까지 보통 얼마나 걸리나요?

상황에 따라 다릅니다.

- 신규 글: 1~2주
- 신생 사이트: 1~2개월
- 내부 링크 많은 글: 1~3일
- 사이트맵만 제출: 2~4주

빨리 하려면:
- 홈페이지에 노출
- 인기 글에서 링크
- SNS/커뮤니티 공유

## 오늘 할 일

### Discovered 상태라면

1. 홈/카테고리/관련 글에서 내부링크 3개 이상 추가
2. sitemap.xml 확인 후 서치콘솔에서 재제출
3. URL 검사에서 색인 요청 1회

### Crawled 상태라면

1. 본문 1,500자 이상 보강 (FAQ 5개 + 단계별 가이드)
2. canonical/리다이렉트 확인해서 중복 URL 정리
3. robots.txt, noindex 재확인

### 공통

서치콘솔 URL 검사로 차단 원인부터 확인하세요.
robots.txt, noindex, canonical 문제는 여기서 한 번에 보입니다.

## 1주일 후에도 안 되면

**추가 확인**
- 서치콘솔 "페이지 색인 생성" 섹션 다시 보기
- "수동 조치" 탭에 패널티 있는지 확인
- Core Web Vitals 점수 확인
- 모바일 친화성 테스트

**도움 받기**
- 구글 서치콘솔 헬프 커뮤니티에 상황 공유
- URL과 증상을 명확하게 설명
- 스크린샷 첨부

**장기 전략**
- 사이트 전체 구조 점검
- 전문가 SEO 진단 고려
- 경쟁사 색인 전략 참고

## 정리하면

**Discovered 상태**
- 내부링크 3개 이상
- 사이트맵 재제출
- 서버 안정성 확인

**Crawled 상태**
- 본문 1,500자 이상
- FAQ 5개 추가
- 중복 URL 정리

**공통**
- robots.txt, noindex 확인
- canonical 확인
- 색인 요청 1회

색인은 기다림의 문제가 아니라 구조와 품질의 문제입니다.
위 방법대로 하나씩 해결하면 대부분 1~2주 안에 색인됩니다.
