**핵심 확인 — 위치 가중(보강 1) 작동 실증**: 4insure.or.kr은 점유율 22.2%로 **구 30% 임계값 로직이었으면 통과했을 것**인데, 판정 근거에 `rank1`이 찍히며 하드킬됐다. 즉 도구가 순위를 실제로 반영한다. → **알바비 ✅를 신뢰 가능**(rank1을 잡아내는 도구가 ✅를 줬다 = alba.co.kr은 top-3 아님). 알바비 빌드 게이트 통과.

**원박스 탐지(보강 3) 작동 실증 — 2차 확인**: `전월세전환율 계산기`가 `네이버 원박스(answer 위젯) — 오가닉 슬롯 매장` 사유로 하드킬됨. 원박스 positive 케이스가 확보되면서, **근무시간계산기에 원박스 경고가 안 뜬 것도 신뢰 가능**(탐지 자체가 되는 도구가 침묵 = 진짜 원박스 없음). 배포된 근무시간계산기의 원박스 리스크 종결.

**층1 판정 미구현 (듀얼테트라 오탐의 원인)** — 보강 6번이 안 들어간 것으로 보임. keyword-radar 2차 보강 요청 항목:
1. **cafe.naver.com 고점유를 노이즈로 무시하지 말 것** — 듀얼테트라 SERP는 `cafe.naver.com 83.3%` + smartstore.naver.com + shoppinglive.naver.com. 카페 80%+는 **공구 열풍의 지문**인데 버킷①(보편 노이즈)로 분류돼 판정에서 빠졌다.
2. **최근 30일 과열 감점 없음** — 신선도 로직이 "90일초과 70%+ 가점"만 있고 반대편이 없다. 듀얼테트라는 최근 30일 글 40%(7일 10% + 30일 30%)인데 무경고. content-strategy 하드게이트 3번("최근 30일 글 30%+면 과열 SERP — 진입 불리")을 감점 규칙으로 구현 필요.
3. **골든 키워드 "공구/공동구매" 킬 시그널** (핵심 변경 2-3) 미반영.
4. **head 브랜드 키워드 판정 제외** — 하드게이트 1번("head 브랜드 키워드 SERP로 판정 금지"). head는 원래 blog 100%라 판정 불가. `head — 진단 롱테일로 재판정 필요` 경고로 처리.

**운용 결론**: 층2(계산기) 판정은 지금 바로 신뢰하고 사용. **층1(트렌드 시드) 판정은 사용 금지** — 재개 시 수동 하드게이트 적용.

## 모바일 우선 계산기 표준 (빌드 시 필수 — 트래픽 70~85% 모바일)

2026-07-12 기존 사이트 모바일 감사 결과 기반. **새 계산기는 이 표준을 그대로 따를 것** (전세계산기가 이미 충족):

- **숫자 입력은 `inputMode="numeric"`(정수)·`"decimal"`(소수)** — 모바일 숫자 키패드. QWERTY 뜨면 실패. (기존 `InputField` 재사용하면 자동 충족)
- **입력 폰트 ≥ 16px** — iOS 포커스 시 자동 확대 방지 (`.calc-input` = 1rem)
- **입력 그리드는 768px에서 1열로 붕괴** — `.calc-inputs { grid-template-columns: 1fr 1fr }` → 모바일 `1fr`
- **넓은 표는 `.calc-table-scroll`(overflow-x auto) 래퍼로 감쌀 것** — 페이지 가로 스크롤 금지, 표 내부만 스크롤. **비교 테이블(`.calc-compare`)도 3열+긴 통화값이면 래퍼 적용** (전세계산기 비교표는 현재 미래퍼 — 새 계산기에선 감쌀 것)
- **터치 타깃 ≥ 44px** — 버튼·입력 패딩 유지
- **설명·FAQ는 계산기 아래 본문에 통합** (짝 글 대신) — 모바일에서 계산기가 먼저, 설명이 아래로
- **AdSense는 `format="auto"` + `full-width-responsive="true"`** 유지 (반응형)

기반 인프라(viewport 메타, theme-color, 헤더 햄버거, 반응형 광고)는 사이트 전역에 이미 적용됨 — 계산기 단위만 위 표준 지키면 됨.

## 확정된 실행 원칙 (이 피벗에서 도출)

- **짝 글 안 씀** — 설명·FAQ·예시는 도구 페이지 본문에 통합(tool+info 한 URL, 체류·viewability↑). 예외: 진단/부작용/차이 앵글이 진짜 있을 때만.
- **승산 최적화 ≠ 수익 최적화** — 근무시간/알바비는 저RPM(검증용). 수익은 고CPC 금융 도구가 우선.
- **키워드 판정 = "권위 인컴번트 유무"** (blog 100%는 보편 노이즈). 저문서수 롱테일 변형만 승산.
- **배포 페이싱** — 에딧(계산기 메타)은 즉시 OK, 새 콘텐츠는 주 3~4편 상한(버스트 금지).
