# 무인 자동화 계획 (Unattended Automation)

매일 글 1편 자동 발행 = "완벽한 무인"으로 가기 위한 설계·실측·고민 포인트 저장소.

## 현재 결정 (2026-07-20)

**자동화 보류 — 트리거는 "블로그 수익 발생".** 자동화 구축 비용/복잡도를 지금 쓸 근거가 약하다. 먼저 **수동 파이프로 계속 발행 → 반응·수익 측정 → 수익이 나기 시작하면 그때 자동화 착수 여부·방식 결정.**
- 지금은 사용자가 Discord "오늘의 키워드"를 보고 직접 골라 이 세션에 넣고 파이프를 돌린다.
- 이 문서는 자동화를 재개할 때 **이어서 고민할 포인트**를 잃지 않기 위한 기록이다.
- 아래 조사(파일럿·엔진·입출력 설계·비용·이미지)는 재개 시 바로 쓸 수 있게 보존.

## Gemini 무료 파일럿 결과 (2026-07-20 실측)

- 모델 `gemini-3.1-flash-lite`, 무료 500회/일. 키는 프로젝트 `.env`의 `GEMINI_API_KEY`.
- 파일럿 하네스: 세션 scratchpad `gemini-pilot.mjs` (임시 — 세션 종료 시 소멸. 결론은 아래에 보존).

| 구멍 | 결과 | 근거 |
|---|---|---|
| ① 의미중복(ledger 대조) | ✅ 해결 | ledger 통째로 주니 4/4 정답. **"초당옥수수 찌는법"을 `chodang-corn-boiling`(삶는법) DUP로 정확 판정** + 크레아틴 로딩·제습기 12리터도 잡음. 무감독 dedup 신뢰 가능(N=4 소표본 단서) |
| ③ 초안 작성 | ✅ 쓸만함 | ~2,000자, frontmatter·3줄요약·존댓말 정상, 자동 룰스캔 위반 0 |
| ② 팩트(단일 출처) | ❌ **블로커** | 무료 티어 `google_search` 그라운딩 **즉시 429(RESOURCE_EXHAUSTED)** — 모델 무관 **계정 레벨 차단**(툴 없는 순수 생성은 200 OK). 그라운딩 없이 조사시키면 **가격·제조사·출처·날짜까지 통째 환각**: 예) "세포랩 앰플 45,000원 (공식몰 정가, 2025-05-20 기준)" — 전부 지어냄. **형식 출처검사도 무력**(출처까지 지어내서) |

**결론:** Gemini 무료 = 판단·글쓰기 O, **사실 X**. 무인에서 팩트는 Gemini에 맡기면 안 됨.

## 작가 엔진 옵션

| 엔진 | GitHub App | 품질 | 비용 | 시크릿 접근 | PC |
|---|---|---|---|---|---|
| **Anthropic 클라우드 Claude 루틴** | 필요 | 최고(그라운딩 검색) | 무료(구독 쿼터) | ❌ 레포만 | 꺼도 됨 |
| GitHub Actions + Gemini | 불필요(내장 `GITHUB_TOKEN`) | Gemini(환각 리스크) | Actions 무료 + Gemini 유료 | ✅ Actions secrets | 꺼도 됨 |
| 로컬 PC cron | 불필요(로컬 git) | Claude/Gemini | 무료/유료 | ✅ `.env` | **켜둬야 함** |

- 루틴은 **Anthropic 클라우드**에서 돌며 구독 쿼터 소모(별도 API 과금 아님, `/loop`과 달리 PC 꺼도 됨).
- 클라우드 루틴은 **로컬 `.env` 접근 불가**(gitignore라 클론에 없음) → `AUTH_SECRET`·`COUPANG_*`·`GEMINI_API_KEY` 못 씀.

## 입력(트리거) 설계 — push 채택

keyword-radar가 이미 오늘의 키워드를 **Discord로 push** 중 → 같은 걸 이쪽으로도 push하면 됨.

```
keyword-radar ──POST──▶ Netlify Function(웹훅) ──GitHub API──▶ 레포 큐(content-queue.json)
```
- **push라서 이쪽에 keyword-radar 키 불필요**(pull이었으면 `AUTH_SECRET` 필요했음).
- 정적 사이트는 POST 못 받음 → **Netlify Function**이 수신자. 하는 일은 "받은 키워드를 큐 파일에 커밋"뿐(상태 없음, 5초).
- 수신자 시크릿 = **GitHub 토큰 하나**(Netlify env) + 웹훅 공유 시크릿(스팸 방지).
- 대안: 수신 즉시 `RemoteTrigger`로 루틴 발사(실시간). 단 큐 방식이 실패내성·기록 우위 → **큐 추천**.

## 출력 설계

- 클라우드 Claude 루틴 채택 시 **GitHub App 필요** — 루틴이 쓴 MDX를 레포에 커밋/PR하려면 Anthropic 클라우드에 레포 쓰기 권한 필요. **입력(push)과 무관한 별개의 문(출력)**.
- **PR-only + 사람 merge** = 무인 커밋 리스크 상쇄 + 이미지 삽입 지점.
- 이미지: frontmatter가 `hero.webp` 참조 → 이미지 없이 main 커밋 시 **sharp 빌드 깨짐** → 이미지 들어올 때까지 **PR 대기 필수**.

## GitHub App 리스크/완화 (검토 완료)

- 리스크: 레포 쓰기 권한을 클라우드에 부여 + **무인 커밋**(네가 안 볼 때 들어옴)이 신규 리스크.
- 완화: **my-blog만** 설치(Only select repositories) · main **branch protection** · PR-only · 사람 merge · **ephemeral 설치 토큰**(PAT보다 안전) · 언제든 uninstall · `.env`는 gitignore라 노출 안 됨.

> **입력/출력은 다른 문이다.** 입력(keyword-radar→레포)은 **push라서 이쪽에 키 불필요**. GitHub App은 **출력**(Claude가 쓴 글을 레포에 커밋/PR)용 — push와 무관하게 필요. "품질 좋은 Claude가 · PC 꺼진 채 · 무료로" 쓰는 것의 유일한 비용이 App 1회 설치.

## 비용 & 이미지 (2026-07-20 확정)

**텍스트 비용** — 경로에 따라 갈림:
- **구독 Routine**: 추가 API 과금 **$0**. Pro/Max 쿼터를 세션처럼 소모. 하루 1편이면 부담 작음
- **Actions + Claude API**: 아래 표 (오늘 5단계 멀티에이전트 기준 대략, 입력 ~300k + 출력 ~30k/편)

| 모델 (100만 토큰당) | 입력/출력 | 1편 | 매일이면 월 |
|---|---|---|---|
| Opus 4.8 | $5 / $25 | ~$2.2 | ~$66 |
| Sonnet 5(프로모) | $2 / $10 | ~$0.9 | ~$27 |
| Haiku 4.5 | $1 / $5 | ~$0.45 | ~$13 |

- 단일 에이전트로 간소화하면 편당 $0.1~0.3까지 내려가나 품질도 하락. 오늘 방식이 비싼 건 5단계로 쪼갠 탓
- **Gemini 무료**: $0 (단 팩트 환각 리스크 — 파일럿 결론 참조)

**이미지** — 자동화 가능·저렴, 품질만 타협:
- 유료 `gemini-3.1-flash-image`(nano-banana 계열) **장당 ~2~4센트** (하루 1장이면 월 $1 미만)
- 이미지 키는 시크릿 있는 곳(**Actions**)에서 호출 — Routine(클라우드 Claude)은 시크릿 없어 이미지 못 만듦. 그래서 **텍스트=Routine, 이미지=Actions** 분리
- 진짜 병목은 비용이 아니라 **품질 큐레이션**: 지금은 사람이 프롬프트 받아 뽑아 고름. 완전 자동이면 그 눈이 빠져 "그럭저럭 AI 히어로"가 나감. 히어로는 장식이라 대개 허용되지만 기준 충족은 실제 뽑아봐야 확인

## "완벽한 무인"까지 남은 고민 포인트 (다음 결정)

1. **이미지 자동화** — 폐기한 자작 스크립트(`generate-hero-image.js`, 품질 미달) 대신 **유료 Gemini 이미지 모델**(`gemini-2.5-flash-image`·`gemini-3.1-flash-image`, nano-banana 계열) 재평가. ⚠️ 이미지 키는 **시크릿 있는 곳(Actions/Netlify)에서 호출**해야 함 — 클라우드 루틴은 시크릿 없음.
2. **팩트 안전 게이트** — 무인 발행 = 사람 눈 0. **YMYL 헬스 위험**(redill 편집사고 전례). 텍스트를 Claude로 쓰면 환각 리스크↓. 초기엔 PR+수동 merge 유지, 신뢰 쌓이면 **저위험(other/tech)만 auto-merge**.
3. **키워드 auto-pick vs human-pick** — 완전무인은 top ✅ 자동선택 + ledger dedup 게이트 필요. 현재는 사람이 Discord 보고 픽.
4. **쿠팡 딥링크** — `COUPANG_*` 키 필요 → 클라우드 루틴 불가. merge 시 로컬 또는 Actions에서 처리.
5. **GitHub App 설치 여부** — 미정. 클라우드 Claude 루틴 경로의 게이트.

## 자동화 재개 시 검증 순서

1. (지금) 수동 파이프 몇 편 더 + 반응 측정
2. 자동화 착수 → **클라우드 Claude 텍스트 파이프부터 검증**: GitHub App 설치 + 큐에 키워드 1개 넣고 일회성 run → "조사→초안→PR" 나오나
3. 되면 **push 입구**(Netlify Function → 큐) 얹기
4. 그다음 **유료 Gemini 이미지** 얹어 마지막 사람 단계 제거
