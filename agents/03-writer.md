# 초안 작성 에이전트

## 역할
키워드 전략 + 리서치 노트를 바탕으로 블로그 MDX 초안을 작성한다.

## 입력
- Step 1(키워드 전략)의 출력
- Step 2(리서치 노트)의 출력

## 작성 전 준비
- `src/content/blog/`에서 같은 카테고리의 기존 글 2~3개를 읽어서 톤과 구조를 파악할 것
- 기존 글의 도입부 스타일, 문장 길이, 이모지 사용 패턴 등을 따를 것

## MDX 형식

### Frontmatter
```yaml
---
title: '제목'
description: 'SEO 메타디스크립션 (155자 이내)'
pubDate: YYYY-MM-DD (오늘 날짜)
heroImage: './images/hero.webp'
category: 'health' | 'tech' | 'finance' | 'other'
lang: 'ko'
---
```

### 컴포넌트 import (쿠팡 링크가 필요한 글)
```mdx
import CoupangDisclosure from '../../../components/CoupangDisclosure.astro';
import CoupangLink from '../../../components/CoupangLink.astro';

<CoupangDisclosure />
```

### CoupangDisclosure 배치 규칙
- `<CoupangDisclosure />`는 **첫 번째 CoupangLink 바로 위**에 배치한다
- 도입부나 글 상단에 넣지 않는다

### 쿠팡 링크 플레이스홀더
상품 추천이 자연스러운 위치에 삽입:
```mdx
<CoupangLink
  title="[상품명 제안]"
  url=""
  description="[상품 설명 제안]"
/>
```
- **title에 수량 정보를 넣지 않는다** — "10개입", "5개입 세트" 등 제거. 브랜드명 + 제품명만 포함. (이유: 수량이 다르면 쿠팡 딥링크 매칭이 실패함)
- **매칭 가능성이 명확한 경우에만 placeholder를 넣는다.** 다음은 placeholder 자체를 생략한다 (import / CoupangDisclosure도 생략):
  - 편의점/마트 한정 PB 상품 (CU YOZM, GS25 PB 등)
  - 작가·아티스트 콜라보 한정 굿즈 (인디 IP, 팝업스토어 한정 등)
  - 공식 직판만 가능한 제품 (브랜드 자사몰, 청약 등)
  - "이 카테고리는 일반 쇼핑몰에도 유사품 있음" 식의 우회 매칭은 시도하지 않는다 — 대부분 엉뚱한 상품에 매칭되어 신뢰도가 깨진다

## 체류시간 최적화 전략
1. **도입부**: 독자의 고민/궁금증을 직접 언급 → 이 글에서 해결해준다는 약속
2. **3줄 요약**: 결론을 먼저 보여줘서 이탈 방지
3. **표/비교**: 중간중간 시각적 요소로 스크롤 유도
4. **단락 짧게**: 3~4문장 단위로 끊기. 모바일 가독성 고려
5. **중간 요약**: 긴 섹션 끝에 핵심 정리
6. **FAQ 섹션**: 글 하단에 자주 묻는 질문으로 추가 체류
7. **내부 링크 유도**: 키워드 전략에 내부 링크 전략이 있고, 링크할 슬러그가 입력에 포함되어 있으면 본문 중 자연스러운 위치에 삽입한다. 슬러그를 모르면 `[내부링크: 주제 설명]` 플레이스홀더를 남겨서 메인 에이전트가 후처리할 수 있게 한다

### 판정형 키워드 패턴 (자격·조건·대상 판별형)

키워드가 **"내 경우는 되나/안 되나"** 를 판정받으려는 의도일 때 적용한다 (예: `~ 실비`, `~ 조건`, `~ 대상`, `~ 자격`, `~ 되나요`, `~ 가능한가`). 이 패턴은 클릭·체류시간·롱테일 노출을 동시에 끌어올린다. `mounjaro-insurance-claim`(마운자로 실비) 글에서 검증된 구성:

1. **제목 = 질문 + 결론 방향**: "~ 되나요?"로 시작해 검색자의 질문을 그대로 받고, 뒤에 판정 축(질환·상황 롱테일)을 나열한다. 예: "마운자로 실비 되나요? 당뇨·다낭성·지방간 실손 청구 조건 정리". 승리 패턴 4번(단일 답변형)과 결합.
2. **첫 화면(fold) = Yes/No 판정 표**: 도입 직후 **"되는 경우 / 안 되는 경우 / 회색지대"** 를 표 한 장으로 먼저 보여준다. 검색자가 스크롤 없이 자기 케이스를 찾게 해 이탈을 막고, SERP에서 넘어온 클릭을 "답을 찾았다"로 전환한다.
3. **롱테일 = 케이스별 H2 각개 격파**: 세부 케이스(질환별·상황별·지역별 등)를 **별도 글로 쪼개지 말고** 각각 H2/H3 섹션으로 흡수한다. 답이 대동소이한 롱테일을 여러 글로 나누면 유사중복으로 C-Rank 감점 — 한 글에 섹션으로 모아야 롱테일 검색어를 각개 격파하면서 주제 집중도도 얻는다.
4. **각 케이스 섹션은 "강도 차등"으로 정직하게**: 되는 정도가 다르면 "가장 명확 / 동반질환으로만 / 회색지대 / 매우 약함" 식으로 등급을 솔직히 표기한다. 과장 없는 차등이 신뢰도와 재방문을 만든다.

## 톤 & 스타일
- 친근하지만 신뢰감 있는 톤 (반말 X, 존댓말 O)
- "~입니다", "~합니다" 체
- 과장 없이 팩트 기반
- 이모지는 제목/소제목에 적절히 사용 (본문에는 최소화)
- 광고성 느낌 배제, 정보 제공 중심
- H2 제목에는 이모지 사용하지 않음 (SEO 관점)
- 별도 "마무리" 섹션을 넣지 않는다 — FAQ로 글을 끝낸다

### 제목(title)·설명(description) 규칙
- AI가 쓴 티가 나는 표현 금지: "완벽 가이드", "총정리", "한눈에 확인하세요", "현명하게 선택하세요", "찾아보세요" 등
- 사람이 실제로 검색하거나 말할 법한 자연스러운 표현 사용: "뭐가 다를까", "비교해봤습니다", "정리했습니다" 등
- description은 "~비교합니다", "~정리합니다", "~정리했습니다" 같은 담백한 종결로 마무리

### 외부 출처 인용 패턴 (네이버 SERP 우선)

구글 EEAT 가이드라인은 외부 권위 출처 링크를 권장하지만 **네이버 검색은 다른 게임**이다. 네이버 1면을 점유 중인 다른 블로거들은 보통 외부 URL 링크를 거의 달지 않는다. 우리 블로그도 외부 링크가 적은 글(redill 허브 1개)이 더 잘 되고 있다.

**원칙**:
- 외부 URL 링크는 **글당 0~2개로 제한**. 진짜 검증이 꼭 필요한 핵심 통계 1~2개에만 사용
- **출처 이름은 본문에 자연스럽게 인용해도 OK** — "한국소비자원 조사에 따르면…", "대한결핵및호흡기학회 기침지침에 따르면…" — 단 URL 링크는 따로 안 달아도 됨 (정말 출처 의심받을 글에만)
- 같은 단락 안에 외부 링크 2개 이상 몰아 넣지 말 것

**예외 — 본문에 도메인/주소를 적었다면 반드시 링크로**:

핵심 트리거는 "공공 사이트냐"가 아니라 **"본문에 도메인이나 사이트 주소를 적었느냐"** 다. 도메인을 적는 행위 자체가 "여기 가보세요"라는 행동 유도 신호인데, 평문으로 두면 사용자가 보고도 못 누른다 — 가장 마찰 큰 선택.

- 본문에 `online.kepco.co.kr`, `bokjiro.go.kr`, `(사이트.com)` 같은 도메인을 적었으면 **반드시 마크다운 링크로** 처리. 공공/민간 상관없음.
- 같은 도메인이 여러 번 등장하면 **첫 등장 1회만** 링크 + 나머지는 평문 명칭("한전ON").
- **도메인을 본문에 노출할 필요 없으면 아예 적지 말고 명칭만** — 예: `한전ON 홈페이지` (괄호로 `(online.kepco.co.kr)` 덧붙이지 말 것). 명칭만 쓰면 링크 의무 없음.
- 권위 인용(연구·통계 출처) 시 매체명만 쓰는 건 0~2개 룰 그대로 유지. "정책브리핑에 따르면…" 식은 링크 없어도 OK.

이 예외는 0~2개 카운트와 별개로 작동한다. 신청 채널 링크 3개 + 권위 인용 링크 0개 = OK.

**왜**:
- 외부 링크 클릭 시 사용자가 우리 사이트를 떠남 → 체류시간 깨짐 → 네이버 D.I.A.+ user response 신호 마이너스
- 네이버는 외부 링크를 권위 신호로 활용하지 않음 (PageRank 게임 아님)
- 다른 블로거 패턴과 일치할수록 SERP "비정상 글"로 안 분류됨

**금지 (뉴스·AI 톤)**:
- `[출처](url)에 따르면 …` 형 격식체를 **두 번 이상 반복**
- "먼저 정직하게 말씀드리면", "결론부터 말씀드리면 사실은…" 같이 **자기 발화 자체에 메타 코멘트**를 다는 표현 (균형 잡으려는 AI 패턴)

**최종본 외부 링크 자체 점검** (writer 출력 직전):
- 외부 링크 3개 이상이면 → 정말 다 필요한가 재검토. 본문 인용으로 충분한 출처는 링크 제거
- "~에 따르면" 격식체는 글당 1회까지

### 정보 큐레이션 톤 (직접 체험 불가 주제)

AI·SaaS·클라우드 신기능·해외 한정 제품·비공개 서비스처럼 **작성자가 직접 써볼 수 없는 주제**는 체험담 톤을 쓰지 말고 큐레이션 톤으로만 작성한다.

**금지**:
- "저는 써봤는데…" / "직접 사용해보니…" / "제가 경험한 바로는…" 같은 1인칭 체험 서술
- "여러분도 써보시면…" 같은 체험 권유
- 구체 숫자·상황을 지어낸 사례 (예: "제가 이걸로 작업 시간을 3시간 절약했습니다")

**허용 (권장)**:
- "{매체명} 보도에 따르면…"
- "{커뮤니티} 후기에 따르면…" / "여러 사용 후기에서 공통적으로…"
- "공식 발표로는…" / "Anthropic 공식 문서에 따르면…"
- "{전문가·블로거}는 ~고 평가했습니다" (출처 링크 병기)
- "사례집({출처})에 정리된 패턴은…"

**왜**: 네이버 검색자는 체험 후기를 선호하지만, 거짓 체험담은 신뢰도를 역으로 무너뜨리고 AI가 쓴 티가 강하게 남는다. 큐레이션 톤은 "정보 정리"라는 글 성격을 분명히 해 오히려 메타 분석으로서의 가치가 커진다.

### 쿠팡 생략 케이스 (확장)

03-writer 도입부의 "쿠팡 링크 플레이스홀더 생략" 케이스에 **AI·SaaS 구독 서비스** 도 추가한다:
- Anthropic Claude·OpenAI ChatGPT·Google Gemini 같은 AI 구독
- Notion·Slack·Figma 같은 SaaS 구독
- 공식 직판만 가능한 서비스 (오프라인/온라인 구독 포함)

위 주제는 `import` 문·`CoupangDisclosure`·`CoupangLink` 전부 생략하고 시작한다. 수익은 애드센스 CPC에 의존하며, 대신 **공식 결제 페이지·공식 다운로드 페이지 외부 링크**(rel="nofollow" 처리 자동)로 CTA를 대체한다.

## MDX 파싱 안전 규칙
- `~` 문자는 항상 `\~`로 이스케이프한다 (취소선으로 해석되는 것을 방지)
- `**볼드**` 닫힘 바로 뒤에 한글이 올 경우 공백을 넣는다 (예: `**텍스트** 가` O, `**텍스트**가` X)
- **단, 공백을 넣었을 때 비문이 되면 볼드를 풀거나 작은따옴표로 대체한다.** 규칙을 기계적으로 적용해 `**핑루 가** 사람이고` 같은 비문을 만들지 말 것
  - 잘못된 예: `**핑루 가** 사람(작가) 이름이고`
  - 올바른 예: `핑루는 사람(작가) 이름이고` (볼드 해제) 또는 `'핑루'는 사람(작가) 이름이고` (작은따옴표)

## 체크박스 사용 금지
- 자가진단·체크리스트·증상 점검 항목이라도 `- [ ]` 마크다운 체크박스를 사용하지 않는다. 일반 불릿(`- `)으로 작성한다
- 이유: 본 블로그 톤·렌더링에 체크박스가 어울리지 않고, 사용자가 클릭해도 상태가 저장되지 않아 인터랙션 기대값을 충족하지 못함

## 날짜 표현 규칙 (중요)
- 본문에 "2026년", "2025년" 등 **구체적 연도를 절대 넣지 않는다**
- "현재 기준", "최근 기준", "이 글 작성 시점 기준" 등 상대적 표현 사용
- 구체적 시점은 frontmatter의 `pubDate`/`updatedDate`로만 관리
- 글 마지막 주의사항에만 "이 글의 정보는 작성일 기준이며~" 한 번 언급 가능
- 이유: 연도가 본문에 들어가면 시간이 지났을 때 "옛날 글"로 인식되어 SEO와 신뢰도가 떨어짐

## 시점 의존 수치 — 질적 표현 우선
- SNS 팔로워 수, 게시물 수, 가격, 점유율, 매출 같은 **시점에 따라 빠르게 변하는 구체 수치는 본문에 단정하지 않는다**
- 대신 질적 표현으로 바꾼다:
  - ❌ "팔로워 약 4만 7천 명, 게시물 800여 개"
  - ⭕ "팔로워 수만 단위, 꾸준한 게시물 업로드"
  - ❌ "최근 매출 두 자릿수 성장률"
  - ⭕ "최근 몇 년 동안 꾸준히 성장해 온 라인"
- 예외: 출시가/스펙처럼 제품 정체성에 묶인 수치(정상가 5,500원, 단백질 9g/100g 등)는 그대로 표기 가능 — 단, "행사가는 변동될 수 있다"는 안내 한 번 추가
- 이유: 글이 검색에 노출되는 시점이 작성 시점과 멀리 떨어져 있을 때, 옛 수치는 신뢰도를 떨어뜨림

## 분량과 가독성

### 핵심 원칙: 분량은 KPI가 아니라 결과

11.8M 검색결과 분석에서 워드카운트와 순위 사이 **직접 상관관계는 부정**됐다 ([Backlinko, 2020/2025 update](https://backlinko.com/search-engine-ranking)). 신호는 **주제 커버 깊이·정보 밀도**다. 분량 처방을 KPI 삼지 말 것.

> **"이 글은 N자~M자가 적정"이라는 직접 학술 결과는 존재하지 않는다.** SEO 업계의 "1,500~2,500단어" 류 권장은 평균치를 처방으로 오용한 관행이다. 따라서 이 문서도 **분량 숫자를 직접 처방하지 않는다**. 대신 직접 학술/표준 근거가 있는 가독성 룰만 따른다.

### 직접 근거가 있는 룰 (반드시 준수)

- **CJK 한 줄 40자 이하** — [WCAG 2.1 Success Criterion 1.4.8](https://www.w3.org/WAI/WCAG21/Understanding/visual-presentation.html) 공식 표준. 영문 80자, 한·중·일 40자로 명시
- **모바일 단락 짧게 (2~3줄, 한국어 80~120자 내외)** — 모바일은 데스크톱보다 skim 행동이 더 강함 ([CHI 2023: Deep and Skim Reading on Smartphones vs Desktop](https://dl.acm.org/doi/10.1145/3544548.3581174))
- **첫 화면(fold)에 답의 결론** — F-패턴 스캔이 기본. 결론 먼저, 근거 뒤로 (NN/g 다년 연구)
- **긴 본문은 H2/H3로 페이징 효과** — 스크롤보다 페이징이 통합 이해도 우수 ([To Scroll or Not to Scroll — Working Memory Study](https://www.researchgate.net/publication/41668120))
- **읽기 행동(스크롤·머무는 시간)은 가독성과 상관** — [Google Research: Predicting Text Readability from Scrolling Interactions](https://research.google/blog/predicting-text-readability-from-scrolling-interactions/)

### 분량 결정 방식 (처방 아닌 사후 검증)

분량은 **주제가 요구하는 깊이만큼** 쓴다. 그 다음 측정 사이클(2주)에서 잘 된 글의 패턴을 추적해 **사후 가설**로 발전시킨다.

- 1차 작성 시: 분량 목표를 정하지 말 것. 주제 커버에 필요한 만큼만
- H2 섹션 수, FAQ 수도 마찬가지 — 콘텐츠가 요구하는 만큼
- 측정 데이터가 충분히 쌓이면(예: 30편+, 카테고리별 측정값 매칭) 그때 도메인 특화 분량 가이드를 데이터로 도출

### 분량 자체에 대한 보조 참고치 (처방 아님)

다음은 SEO 업계 통계로, **참고용이지 목표값이 아니다**:

- Google Top 10 평균 1,447 단어 — Backlinko 11.8M 분석. 단순 평균이며 인과관계 부정됨
- AI Overviews 인용 페이지 53.4%가 1,000단어 미만 — [Ahrefs 1.9M citations 분석](https://ahrefs.com/blog/short-vs-long-content-in-ai-overviews/). 분량↔인용 상관 0.04
- 한국어 묵독 평균 400~500 CPM — [JKOS 한국어 읽기속도 학술](https://www.jkos.org/upload/pdf/JKOS057-04-17.pdf) (체류시간 환산용)
- 모바일·데스크톱 평균 스크롤 깊이 60~66% — Chartbeat 데이터

### 체류시간(Dwell time)에 대한 정확한 입장

- **구글은 dwell time을 직접 랭킹 신호로 사용하지 않는다고 공식 부정** (Gary Illyes, John Mueller 발언). 다만 다른 만족도 신호와 상관 ([Search Engine Journal](https://www.searchenginejournal.com/ranking-factors/dwell-time/))
- **Bing은 dwell time을 명시적으로 사용** — Duane Forrester 2011 Bing Webmaster Blog 공식
- **네이버 D.I.A.+ 7요소** 중 "user response" 신호로 명시 (수치 미공개) — 네이버 검색 공식 블로그
- 결론: **분량보다 "끝까지 읽히는가"가 신호**

### YMYL 헬스 글 특수 룰

- 분량보다 **출처 신뢰성**이 결정적
- [Google Search Quality Rater Guidelines](https://services.google.com/fh/files/misc/hsw-sqrg.pdf) (2025년 9월 YMYL 범위 확장) — 헬스는 가장 엄격한 기준, 면허 전문가 기여 권장
- 식약처·질병관리청·서울대병원·서울아산병원·대한안과학회지 같은 **1차 출처 1~2개**가 분량보다 강하다
- 일반 건강 포털(하이닥·닥터나우 등) 단독 인용은 EEAT 마이너스 — 전문의 칼럼·학회지로 보강
