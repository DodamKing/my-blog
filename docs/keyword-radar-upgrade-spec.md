## 배경 — 왜 고쳐야 하나

keyword-radar는 키워드별로 **검색량 / 경쟁강도 / 기회지수 / 문서수 / 경쟁률 / 상위 노출 도메인 / 발행일 분포 / "블로그 진입 판단"**을 뽑아준다. 실사용(blog.dimad.kr, 네이버 검색 대상 외부 도메인)에서 **치명적 약점**이 드러났다:

- **"블로그 진입 판단"이 거의 항상 "◎ 진입 좋음"으로 뜬다.** 원인: `blog.naver.com`이 **모든 키워드에서 상위 노출 100%**로 등장하기 때문. 이 지표는 사실상 상수라 **판별력이 0**이다.
- 그 결과 **못 이기는 키워드와 이길 수 있는 키워드를 도구가 구분하지 못한다.** 예:
  - `4대보험계산기`: 상위에 **4insure.or.kr(공식 4대사회보험 계산기)** 점유 → 외부 도메인은 못 이김. 그런데 도구는 "쉬움 / 기회지수 99.6 / 진입 좋음"으로 표시.
  - `근무시간계산기`: 상위가 블로그 + 무명 소형 툴사이트뿐, 공식 인컴번트 없음 → 이길 수 있음. 도구 표시는 위와 **똑같이** "진입 좋음".
- 즉 **"경쟁강도 쉬움" 라벨은 문서수 기반 착시**이고, 외부 도메인 관점의 실제 진입 가능성을 반영하지 못한다.

핵심 통찰: **blog/cafe/지식iN 등 네이버 UGC 점유는 보편 노이즈다. 진짜 판별선은 "권위 있는 전용 인컴번트(정부·공식기관·대형 플랫폼)가 SERP에 있느냐"이다.** 이걸 자동으로 판정하게 만드는 게 이 업그레이드의 핵심.

## 핵심 변경 1 — 상위 노출 도메인 3분류 classifier

이미 **상위 노출 도메인 리스트는 수집하고 있으므로**, 그 도메인들을 아래 3버킷으로 자동 분류하는 레이어만 추가한다.

| 버킷 | 정의 | 예시 도메인 | 점수 영향 |
|---|---|---|---|
| **① 노이즈 (무시)** | 모든 키워드에 보편 등장하는 네이버 UGC | `blog.naver.com`, `m.blog.naver.com`, `cafe.naver.com`, `in.naver.com`, `kin.naver.com`, `post.naver.com` | 점수 반영 안 함 |
| **② ⛔ 권위 인컴번트 (킬/대폭 감점)** | 외부 도메인이 못 이기는 공식·대형 | `*.go.kr`(moel.go.kr 등), `terms.naver.com`(지식백과), 공식 기관 계산기(`4insure.or.kr`), 대형 플랫폼(`saramin.co.kr`, `jobkorea.co.kr`, `alba.co.kr`, `albamon.com`), 은행/금융사 공식몰, **네이버 자체 계산기 위젯(원박스)** | 등장(특히 점유율 30%+) 시 기회지수 대폭 하향 |
| **③ ✅ 소형 툴/콘텐츠 (감점 없음)** | 나머지 무명 계산기·툴·콘텐츠 도메인 | `miniwebtool.com`, `calculatorlib.com`, `tools.devcomma.com`, `4th.kr`, `it-b.co.kr`, `albabee.com`, `refininfo.com` 등 | 없음 (우리가 이길 수 있음) |

**판정 규칙:**
- 상위 노출 도메인에 **버킷②가 점유율 30%+ 로 존재** → **`⛔ 인컴번트 점유 — 어려움`** 플래그, 기회지수 대폭 하향.
- 버킷② 없고 버킷③만 있으면 → **`✅ 틈 있음 — 진입 가능`** 플래그.
- 기존 "블로그 진입 판단" 필드는 버킷①이 보편이라 무의미 → **위 플래그로 대체(또는 병기)**.

> 버킷② 도메인 목록은 하드코딩 시드 리스트 + 패턴(`*.go.kr`, `*.or.kr` 중 공식기관)으로 시작하고, 운영하며 누적 확장한다.

## 핵심 변경 2 — 추가 신호 3개

1. **head 레드오션 경고**: 문서수 절대값이 크면(예: 10,000+) "경쟁강도 쉬움" 라벨이어도 **`head — 인컴번트/포화 위험`** 경고. (실측: 퇴직금 계산기 문서 20,306 / 연봉계산기 38,573 — 전부 정부·대형플랫폼 점유)
2. **신선도 갭 가점**: 발행일 분포에서 **90일초과 비율이 높으면(70%+)** `신선도 갭 — 신규 진입 유리` 가점. (상위가 낡은 글 위주면 새 글이 치고 들어갈 여지)
3. **공구/공동구매 킬 시그널** (트렌드 시드 발굴용): 시드의 골든 키워드에 **"공구 / 공동구매"**가 뜨면 = 제휴 수익 존재 = 체험단·블로그가 쏟아지는 신호 = 규제 해자 없음 → 시드 감점. (실측: 홈뷰티 "듀얼테트라" 골든이 전부 "프로/공구" → 폐기)

## 핵심 변경 3 — 독립 심사 반영 (착수 전 필수, 2026-07-12)

독립 리뷰에서 발견된 6개 보강. 반영 안 하면 자동화 판정 함수로 쓸 수 없다.

1. **위치 가중 (position-weighted) — 버킷② top-3면 %무관 하드킬.** 30% 임계값은 자의적·위치맹목. rank1의 공식 계산기 하나가 rank8~10 소형툴 셋보다 치명적이다. 도구가 이미 순위를 갖고 있으니 **버킷② 도메인이 상위 3위 안에 하나라도 있으면 점유율 무관 하드킬.**
2. **하드코딩 리스트 → 크로스-키워드 권위 휴리스틱 (가장 큰 누락).** 하드코딩은 신규 인컴번트(새 공식 계산기·은행 신규 툴)를 미탐 → 거짓 "진입 가능"(위험한 오탐). `*.go.kr` 패턴도 saramin·jobkorea·alba 같은 `.co.kr` 인컴번트는 못 잡는다. **해결: "이 도메인이 여러 다른 키워드의 1면 상위에 반복 등장하는가?"를 도구 자체 축적 데이터로 계산 → 반복 상위 도메인을 플랫폼 인컴번트(버킷②)로 자동 승격.** 하드코딩 리스트는 초기 시드로만.
3. **네이버 자체 계산기 원박스 = 감점 아니라 하드킬.** 원박스가 뜨면 오가닉 슬롯이 매장된다. 이진 판별 가능하니 버킷②에서 최상위 킬 신호로 격상.
4. **모바일 SERP 기준으로 판정.** 페르소나 모바일 70~85%인데 PC SERP로 판정하면 오판. 모바일은 결과 수 적고 네이버 위젯이 더 많이 떠서 더 불리하다. **classifier 입력은 모바일 SERP.**
5. **버킷③ "감점 없음" → 중립~약주의 + 신선도 반영.** calculator.io·miniwebtool처럼 성숙·SEO 강한 대형 툴이 이미 1위면 실질 경쟁자다. 버킷③도 그 툴의 신선도(낡음/얇음)를 반영해 가감.
6. **층1(트렌드 시드) 테스트 케이스 추가.** 현재 케이스 5개가 전부 계산기(층2). 공구 킬 시그널의 pass/kill 케이스가 없다. 공구 시그널은 "층1 파이프라인" 라벨을 명시할 것.

## 최종 목표 — API 자동화의 판정 함수 ✅ 달성 (2026-07-16)

이 3분류 + 추가 신호 로직은 그 자체로 **시드→조회→SERP 판정 루프의 판정 함수**다. keyword-radar가 이 판정 결과를 API로 노출하면, blog.dimad.kr 쪽에서 Claude가 **시드 발굴 → 키워드 조회 → 진입 가능성 자동 판정 → 큐레이션**을 무인으로 돌릴 수 있다 (content-strategy.md "프로그램화 로드맵" 참조). 즉 이 업그레이드는 수동 SERP 확인 노동을 없애는 동시에 자동화의 전제 조건이다.

### 달성 결과

`POST /api/judge` 가 `judgeEntry()` 를 서버에서 호출해 `verdict`(hard/soft/open) + `reasons` + `incumbents` + `signals` 를 내려준다. 판정 신호(onebox·문서수·발행일)를 서버가 모으므로 **호출 1번**이면 되고, 임계값·도메인 분류를 blog 쪽에서 재현할 필요가 없다 — **판정의 단일 출처가 서버**라 로직이 바뀌어도 도구 UI와 blog가 함께 따라간다.

이로써 루프 4단계가 모두 API로 연결됐다: 시드 발굴(`/api/expand`) → 조회(`/api/analyze`) → 진입 판정(`/api/judge`) → 큐레이션(Step 1 에이전트). 사용자가 도구를 돌려 결과를 복붙하는 노동이 사라졌다.

**실측으로 확인된 착시 해소** (2026-07-16, 이 문서가 겨냥한 바로 그 문제):

| 키워드 | 기존 데이터 라벨 | `/api/judge` 판정 |
|---|---|---|
| 크레아틴 | 기회점수 **97.7** / "쉬움" | **⛔ hard** — 원박스 + 나무위키·지식백과 점유, head 포화(74,226) |
| 실업급여 조건 | — | △ soft — easylaw.go.kr·work24.go.kr 혼재 |
| 알바비계산기 | 기회점수 88.9 / "쉬움" | ✅ open — 강자 소수, 문서 1,416 |

크레아틴이 정확히 **"쉬움 라벨 착시"의 표본**이다. 검색량(79,300)이 커서 문서 74,226건에도 경쟁률이 0.94로 낮게 계산됐을 뿐, 1면은 블로그가 비집을 자리가 없다. 데이터 점수만 봤다면 최우선 공략 대상으로 잡았을 키워드다.
