## 백로그 (우선순위 순)

1. **[결정 대기]** 목표 프레이밍 — $10~100/일 야망 확정 시, 네이버 계산기 외 **별도 채널(구글) 실현성 검토**를 백로그 상위로.
2. **[✅ 빌드 완료]** 근무시간계산기 — 레지스트리 자동화 + 순수 산술 계산기 + 설명 통합 완료. 남은 것: **모바일 원박스 확인 → 배포 → 측정**. 다음 노동 도구는 **알바비계산기**(레지스트리 1줄 + 페이지).
3. **[빌드/검증]** 금융 계산기(중도상환수수료·전월세 전환율) — SERP 실측 후. **수익 타겟(고RPM $6~7)**. 근무시간 검증 완료 후 스케일.
4. **[배포]** 전세계산기 수정 반영 — build + deploy, 2주 측정.
5. **[별도 레포]** keyword-radar 6개 보강 구현 (`keyword-radar-upgrade-spec.md`). 완료 시 시드→조회→판정 루프 자동화 가능.
6. **[주간]** 층1 트렌드 시드 스캔 — 흡입/복용형 규제 신제품(부작용 검색 자연발생, 공구 안 붙는 것). 클린 시드 뜰 때만 발행.
7. **[야망 엔진]** $100/일 경로 — 계산기 스윗 확장(엔진1) + 층1 트렌드 파이프라인(엔진2) + 비-네이버 채널(엔진3). 단일 엔진 천장 $2~4/일이므로 병렬 필수.

## 모바일 우선 계산기 표준 (빌드 시 필수 — 트래픽 70~85% 모바일)

2026-07-12 기존 사이트 모바일 감사 결과 기반. **새 계산기는 이 표준을 그대로 따를 것** (전세계산기가 이미 충족):

- **숫자 입력은 `inputMode="numeric"`(정수)·`"decimal"`(소수)** — 모바일 숫자 키패드. QWERTY 뜨면 실패. (기존 `InputField` 재사용하면 자동 충족)
- **입력 폰트 ≥ 16px** — iOS 포커스 시 자동 확대 방지 (`.calc-input` = 1rem)
- **입력 그리드는 768px에서 1열로 붕괴** — `.calc-inputs { grid-template-columns: 1fr 1fr }` → 모바일 `1fr`
- **넓은 표는 `.calc-table-scroll`(overflow-x auto) 래퍼로 감쌀 것** — 페이지 가로 스크롤 금지, 표 내부만 스크롤. **비교 테이블(`.calc-compare`)도 3열+긴 통화값이면 래퍼 적용** (전세계산기 비교표는 현재 미래퍼 — 새 계산기에선 감쌀 것)
- **터치 타깃 ≥ 44px** — 버튼·입력 패딩 유지
- **설명·FAQ는 계산기 아래 본문에 통합** (짝 글 대신) — 모바일에서 계산기가 먼저, 설명이 아래로
- **AdSense는 `format="auto"` + `full-width-responsive="true"`** 유지 (반응형)

기반 인프라(viewport 메타, theme-color, 헤더 햄버거, 반응형 광고)는 사이트 전역에 이미 적용됨 — 계산기 단위만 위 표준 지키면 됨.

## 확정된 실행 원칙 (이 피벗에서 도출)

- **짝 글 안 씀** — 설명·FAQ·예시는 도구 페이지 본문에 통합(tool+info 한 URL, 체류·viewability↑). 예외: 진단/부작용/차이 앵글이 진짜 있을 때만.
- **승산 최적화 ≠ 수익 최적화** — 근무시간/알바비는 저RPM(검증용). 수익은 고CPC 금융 도구가 우선.
- **키워드 판정 = "권위 인컴번트 유무"** (blog 100%는 보편 노이즈). 저문서수 롱테일 변형만 승산.
- **배포 페이싱** — 에딧(계산기 메타)은 즉시 OK, 새 콘텐츠는 주 3~4편 상한(버스트 금지).
